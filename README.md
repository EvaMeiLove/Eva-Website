# Персональная страничка (link-in-bio)

Милый одностраничный сайт в розово-пастельных тонах: профиль, соцсети, галерея, музыка с YouTube Music, видео с YouTube. Контент и аналитика настраиваются в одном файле — **`src/config/site.config.ts`**. Сборка готова для **Netlify** + **GitHub**.

## Как это работает

1. **`src/app/page.tsx`** собирает главную страницу: подключает секции по флагам из `site.config.sections`.
2. **`site.config.ts`** — единственное место, где задаётся текст, ссылки, списки фото/треков/видео, SEO-заголовки и **опционально** аналитика.
3. После пуша в ветку **`main`** на GitHub **Netlify** запускает `npm run build` и публикует новую версию (если сайт уже привязан к репозиторию).

**Секции на главной (сверху вниз):**

| Секция | Компонент | Поведение |
|--------|-------------|-----------|
| Профиль | `Hero.tsx` | Аватар, имя, ник, слоган, bio, бейджи. Если аватар не загрузился — розовая заглушка. |
| Ссылки | `SocialLinks.tsx` | Карточки соцсетей по массиву `links`. |
| Галерея | `Gallery.tsx` | Горизонтальная карусель (много фото — страница не «растягивается»). Миниатюры **3:4**, обрезка `cover`. Клик — лайтбокс в **полном размере** без обрезки. |
| Музыка | `YouTubeMusicSection.tsx` + `YouTubeMusicPlayer.tsx` | На сервере запрашивается oEmbed YouTube (название, автор, обложка), кеш ~сутки. Плеер: один трек на экране, стрелки и список; длинный список прокручивается внутри блока. |
| Видео | `VideoSection.tsx` | Один плеер, переключение **← / →** и список роликов. YouTube или локальный `mp4`. |
| Подвал | `Footer.tsx` | Имя внизу страницы. |

Локальный аудиоплеер (`AudioPlayer.tsx`) в проекте есть, но по умолчанию **выключен** (`sections.audio: false`). Музыка идёт через **YouTube Music** по `youtubeId`.

## Настройка контента (`site.config.ts`)

Открой **`src/config/site.config.ts`**. Комментарии в файле подсказывают, что менять.

### Профиль и SEO

- **`profile.name`**, **`nickname`**, **`tagline`**, **`bio`**, **`badges`**
- **`profile.avatar`** — путь, например `"/images/avatar.jpg"` или `""`, если без фото
- **`profile.siteTitle`**, **`profile.siteDescription`** — вкладка браузера и сниппет для соцсетей/поиска

### Ссылки (`links`)

Каждый элемент: `id`, `label`, `url`, `icon`, `color`, `username`, `description`.

Иконки: `send` (Telegram), `instagram`, `youtube`, `twitter`, `tiktok`, `reddit`, `pinterest`, `discord`, `github`, `vk`, `globe`, `link`.

### Галерея (`gallery`)

Массив объектов: уникальный **`id`**, **`src`** (от корня сайта, обычно `/images/...`), **`alt`**, **`caption`**.

Пустой массив `gallery: []` — секция не показывается (если `sections.gallery: true`, лучше выключить секцию или оставить массив непустым).

### YouTube Music (`youtubeMusicTracks`)

Из ссылки вида `https://music.youtube.com/watch?v=XXXX` или `youtube.com/watch?v=XXXX` возьми **`XXXX`** — это **`youtubeId`**.

```ts
youtubeMusicTracks: [
  { id: "1", youtubeId: "nb1qRwEe0mw" },
  { id: "2", youtubeId: "8NoFd-ruKws" },
],
```

Название, исполнитель и превью подтягиваются с YouTube (oEmbed). У каждого трека свой уникальный **`id`**.

### Видео (`videos`)

**YouTube:**

```ts
{ id: "1", title: "Заголовок", type: "youtube" as const, youtubeId: "WiJ5uVVoGEw" },
```

**Локальный файл** (положи файл в `public/video/` или другой путь в `public/`):

```ts
{ id: "2", title: "Клип", type: "local" as const, src: "/video/clip.mp4" },
```

### Видимость секций (`sections`)

```ts
sections: {
  gallery: true,
  audio: false,              // локальные mp3 — по умолчанию выкл.
  youtubeMusicTracks: true,
  videos: true,
},
```

Если **`youtubeMusicTracks`** или **`gallery`** включены, но массив **пустой**, соответствующий блок на странице не появится (в `page.tsx` есть проверки `length > 0`).

### Аналитика (`analytics`)

```ts
analytics: {
  googleSiteVerification: "",  // meta Google Search Console; пусто = не выводится
  yandexMetrikaId: "",         // только цифры счётчика; пусто = скрипт Метрики не грузится
},
```

Значения подставляются в **`src/app/layout.tsx`**. Чужие счётчики сюда не вставляй при копировании проекта другу — пусть у неё свои или пустые строки.

## Медиа: куда класть файлы

| Назначение | Папка | Путь в конфиге |
|------------|--------|----------------|
| Аватар, фото галереи | `public/images/` | `/images/имя.jpg` |
| Локальные mp3 (если включишь `audio`) | `public/audio/` | `/audio/имя.mp3` |
| Локальное видео | `public/video/` или любая подпапка `public/` | `/video/имя.mp4` |

Имена файлов **чувствительны к регистру** на серверах Linux (Netlify).

## Первый деплой: GitHub + Netlify

1. **GitHub:** создай репозиторий, залей код (через GitHub Desktop, `git push`, или загрузкой файлов).
2. **Netlify:** войди → **Add new site** → **Import an existing project** → выбери **GitHub** и нужный репозиторий.
3. Настройки сборки обычно подхватываются из **`netlify.toml`**:
   - команда: `npm run build`;
   - для Next.js на Netlify используется совместимый рантайм (при необходимости в панели Netlify включи плагин / пресет **Next.js**, если сборка попросит).
4. После успешного деплоя Netlify выдаст **URL** вида `https://что-то.netlify.app`. Домен можно сменить в настройках сайта.

## Как обновлять сайт

**Через GitHub (без программ на компьютере):**

1. Открой в репозитории файл `src/config/site.config.ts`.
2. **Edit** (карандаш) → внеси правки → **Commit changes** в ветку `main`.
3. Подожди 1–3 минуты и обнови сайт с полным обновлением страницы (Ctrl+F5).

**Локально (Cursor / VS Code + Git):**

```bash
git add .
git commit -m "обновила контент"
git push
```

Фото: **Upload files** в `public/images/` на GitHub, затем добавь записи в `gallery` и сделай commit.

## Скопировать шаблон другу

Подойдёт **архив** (GitHub → **Code** → **Download ZIP**) или **новый репозиторий** с копией файлов.

Пусть друг:

1. Скопирует проект к себе и откроет в редакторе.
2. Полностью заменит данные в **`site.config.ts`** (профиль, ссылки, галерея, треки, видео, `siteTitle` / `siteDescription`).
3. В **`analytics`** поставит **пустые строки**, если не нужна твоя Метрика и Google-верификация.
4. **Удалит** из `public/` файл **`google*.html`**, если он был у тебя — он привязан к чужому сайту в Search Console.
5. Заменит **свои фото** в `public/images/`.
6. Создаст **свой** репозиторий и подключит **свой** сайт на Netlify.

Так у каждого свой счётчик и своя верификация, без пересечений.

## Аналитика и Search Console

- **Яндекс.Метрика:** номер счётчика в `analytics.yandexMetrikaId`. Пустая строка — скрипт не подключается.
- **Google Search Console:** строка верификации в `analytics.googleSiteVerification` → в `<head>` попадёт meta-тег. Дополнительно в GSC можно загрузить **HTML-файл** — положи его в **`public/`**, закоммить; имя файла должно совпадать с тем, что выдал Google.

## Проверка перед публикацией

- Все пути к картинкам и файлам существуют в `public/`.
- У каждого элемента в `gallery`, `youtubeMusicTracks`, `videos` уникальный **`id`**.
- Для YouTube везде корректный **`youtubeId`** (11 символов из ссылки после `v=`).
- `sections` отражает то, что хочешь показать.

## Частые вопросы

**Сайт не обновился после пуша**  
Подожди пару минут, зайди в Netlify → **Deploys** и посмотри, зелёный ли последний деплой. Иногда помогает жёсткое обновление страницы (Ctrl+F5).

**Картинка 404 или не та**  
Проверь имя файла и регистр букв. Путь в конфиге начинается с **`/`** (например `/images/photo.jpg`).

**Обложка трека YouTube не грузится**  
Редко бывает ограничение CDN; в коде есть заглушка. Проверь, что `youtubeId` верный.

**Гидрация / ошибки в консоли**  
В проекте уже убраны источники рассинхрона (например случайные значения на сервере и клиенте). Если добавляешь свой код — не используй `Math.random()` в разметке без `useEffect`.

**Нужен другой хостинг**  
Проект — обычный Next.js: подойдёт Vercel или любой Node-хостинг с `next build` / `next start`. Для Netlify ориентируйся на актуальную документацию Next + Netlify.
